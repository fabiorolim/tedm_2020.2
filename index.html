Requisitos mínimos do projeto:


  • O projeto deverá ser composto por uma aplicação front-end e uma aplicação back-end.
  • As aplicações devem se comunicar através de uma API REST.
  • O front-end deverá ser desenvolvido em react-native ou outro framework/biblioteca de sua preferência, exceto plataformas no-code.
  • O back-end deverá ser desenvolvido em Django ou outro framework/biblioteca de sua preferência, exceto plataformas no-code.
  • O projeto deverá apresentar minimamente um CRUD.



Avaliação do projeto


  • Cada projeto será avaliado conforme o tamanho da equipe.
  • Cada pessoa em uma equipe equivale a 10 pontos Ex: time 1 tem 4 pessoas, logo receberá até 40 pontos.
  • Fica sob a responsabilidade dos membros do time dividir o total de pontos dado ao projeto entre cada membro, contabilizando assim sua nota final individual.
  • Nos casos em que não houver consenso do time sobre a divisão dos pontos, a pontuação do projeto será reduzida em 20% e dividida igualitariamente pela quantidade de membros.
  • Os critérios de pontuação serão os seguintes:
    1. Apresentação do projeto (10%)
    2. Organização dos repositórios GitHub (20%)
    3. Organização do código (20%)
    4. Deploy do back-end (20%)
    5. Funcionalidades mínimas atendidas (30%)
